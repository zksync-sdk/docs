## Enumerations

### `AccountAbstractionVersion`

::field-group
  ::field{name="None" type="0"}
  Used for contracts that are not accounts.
  ::
  ::field{name="Version1" type="1"}
  Used for contracts that are accounts.
  ::
::

### `AccountNonceOrdering`

::field-group
  ::field{name="Sequential" type="0"}
  Nonces should be ordered in the same way as in externally owned accounts (EOAs). This means, for instance, that
  the operator will always wait for a transaction with nonce `X` before processing a transaction with nonce `X+1`.
  ::
  ::field{name="Arbitrary" type="1"}
  Nonces can be ordered in arbitrary order.
  ::
::

## Types

### `Address`

0x-prefixed, hex-encoded, Ethereum account address as a string.

### `BalancesMap`

Type defining a map object containing accounts and their balances.

- `{ [key: string]: bigint }`

### `BlockTag`

Pipe-delimited list of block labels that includes block number in binary and hex plus block statuses.

- `BigNumberish`
- `string` // block hash
- `committed`
- `finalized`
- `latest`
- `earliest`
- `pending`

### `PaymasterInput`

Type definition for a paymaster input specified as either approval-based or general.

- `ApprovalBasedPaymasterInput` | `GeneralPaymasterInput`

### `Signature`

0x-prefixed, hex-encoded, ECDSA signature as a string.
